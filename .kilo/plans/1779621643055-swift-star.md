# Plan: University & Major Management APIs

## Codebase Analysis

### Tech Stack
- **Runtime**: Node.js + TypeScript + Express 5.x
- **Module system**: `commonjs` (tsconfig: `"module": "commonjs"`, `"esModuleInterop": true`)
- **DB**: MySQL via `mysql2/promise` connection pool
- **Auth**: JWT (`jsonwebtoken`) + `bcryptjs`
- **Validation**: `express-validator` in routes, `validationResult()` checked in controllers
- **Docs**: `swagger-ui-express` + `swagger-jsdoc`

### Existing Architecture (no service layer, no DTOs)
```
src/
├── types/index.ts       → Shared TS interfaces (User, CandidateProfile, ApiResponse…)
├── constants/auth.ts    → Error/message string constants
├── config/
│   ├── database.ts      → mysql2 Pool export as `default`
│   └── swagger.ts       → OpenAPI spec object
├── middleware/
│   ├── auth.middleware.ts → authenticate / authorize guards
│   └── error.middleware.ts → AppError, errorHandler, notFound
├── models/              → Static-class data layer (direct pool.execute SQL, NO ORM)
├── controllers/         → HTTP handler functions (direct res.status().json(), no AppError throw)
├── routes/              → Express Router with express-validator chains
└── database/
    ├── migrate.ts       → All DDL in one file, sequential execute, process.exit
    └── seed.ts          → Simple INSERT IGNORE seed data
```

### Key Conventions (strictly enforced)
| Convention | Exact pattern observed |
|---|---|
| Response shape | `{ success: boolean, message: string, data?: T, errors?: unknown[] }` |
| Delete strategy | **Soft delete**: set `status = 'INACTIVE'` (not hard DELETE); return boolean confirming row was found |
| List response | `{ success, data: [...], pagination: { total, page, limit, totalPages } }` |
| Status enums | `ACTIVE` / `INACTIVE` — consistent across `universities`, `majors`, `admission_combinations`, `major_combinations` |
| Controller fn names | camelCase: `getUsers`, `getUserById`, `updateUser`, `deleteUser` |
| Controller error pattern | Direct `res.status(code).json({ success: false, message: '...' })` then `return` — never throws `AppError` |
| Model class pattern | `static async findAll`, `static async findById`, `static async create`, `static async update`, `static async delete` — parameterized SQL via `pool.execute<RowDataPacket[]>()` |
| Row result unwrap | `rows.length ? (rows[0] as Type) : null` |
| Update all fields | Builds `SET col = ? ...` dynamically; always appends `updated_at = NOW()` |
| Delete all rows | Builds `DELETE FROM ... WHERE id = ?` returning `affectedRows > 0` boolean |
| Route ordering | `router.use(authenticate)` at top; `authorize()` per-route for ADMIN routes |
| Validation | `express-validator` chains in `routes/*.ts`; `validationResult()` checked in controller at start |
| Messages | Vietnamese strings, single quotes for keys |
| i18n constants file | Separate `<resource>_ERRORS` and `<resource>MESSAGES` objects in `src/constants/` |
| DTOs | **NONE** in the current codebase — skip DTO layer entirely |
| Import order | `express` types first → third-party → internal `../…` |
| Indentation | 2-space indent |
| String quoting | Single quotes for JS strings |
| Plural folder names in route path | `router.use('/api/universities', ...)` |
| Swagger | One file `config/swagger.ts`, flat `paths` object, `as const` assertion |
| Response type annotations | `Promise<void>` on all controller exports |
| Pagination | `page` and `limit` from `req.query`, both coerce with `Number()` |
| tsconfig | `strict: true`, `declaration: true`, `declarationMap: true`, `sourceMap: true`, target ES2020 |

## New Files to Create

### ⚠️ CRITICAL ADJUSTMENT — `majors` table mismatch with spec
The existing migration (`migrate.ts:65-78`) defines `majors` with only `id, university_id, code, name, description, status, created_at, updated_at`. The **spec requires** `admission_combinations_id` (FK → `admission_combinations`) and `min_score` (DECIMAL(4,2)). These must be added to the migration before any Major endpoint using them can work.

**`UniversityModel` methods:**
- `create(data)` — INSERT, return inserted row
- `findAll(page, limit)` — SELECT + COUNT, return `{ universities, total }`
- `findById(id)` — SELECT WHERE id, return `University | null`
- `existsByCode(code, excludeId?)` — SELECT 1 WHERE code = ? AND (id != ? OR ? IS NULL)
- `update(id, data)` — dynamic SET + updated_at = NOW(), re-fetch via findById
- `softDelete(id)` — UPDATE SET status = 'INACTIVE' WHERE id = ?, return boolean
- `hasRelatedApplications(universityId)` — SELECT 1 FROM applications WHERE university_id = ? LIMIT 1

**`MajorModel` methods:**
- `create(data)` — INSERT, return inserted row
- `findAllByUniversity(universityId, page, limit)` — SELECT + COUNT with WHERE university_id = ?
- `findById(id)` — SELECT WHERE id, return `Major | null`
- `existsByCode(universityId, code, excludeId?)` — SELECT 1 WHERE university_id = ? AND code = ? AND (id != ? OR ? IS NULL)
- `update(id, data)` — dynamic SET + updated_at = NOW(), re-fetch via findById
- `softDelete(id)` — UPDATE SET status = 'INACTIVE' WHERE id = ?, return boolean
- `hasRelatedApplications(majorId)` — SELECT 1 FROM applications WHERE major_id = ? LIMIT 1
- `universityExists(universityId)` — SELECT 1 FROM universities WHERE id = ? LIMIT 1
- `admissionCombinationExists(admissionId)` — SELECT 1 FROM admission_combinations WHERE id = ? LIMIT 1

### 4. `src/controllers/university.controller.ts`
10 exported async functions, each annotated `Promise<void>`, each starting with `validationResult` check:

```typescript
import { Request, Response } from 'express';
import { validationResult } from 'express-validator';
import { UniversityModel, MajorModel } from '../models/university.model';
import { UNIVERSITY_ERRORS, UNIVERSITY_MESSAGES, MAJOR_ERRORS, MAJOR_MESSAGES } from '../constants/university';
```

Each function:
1. Checks `validationResult(req)` → 400 if invalid
2. Calls model, catches 409 ER_DUP_ENTRY→ 409
3. Handles 404 not-found → 404
4. Handles 409 in-use → 409
5. Returns `res.status(code).json({ success: boolean, message, data? })` then `return`

**`createUniversity`**  — POST `/api/universities`
```typescript
const { code, name, address, phone, email, website, description } = req.body;
if (await UniversityModel.existsByCode(code)) {
  res.status(409).json({ success: false, message: UNIVERSITY_ERRORS.CODE_EXISTS });
  return;
}
const university = await UniversityModel.create({ code, name, address, phone, email, website, description });
res.status(201).json({ success: true, message: UNIVERSITY_MESSAGES.CREATED, data: university });
```

**`updateUniversity`**  — PUT `/api/universities/:id`
```typescript
const { code, name, address, phone, email, website, description, status } = req.body;
const university = await UniversityModel.findById(Number(req.params.id));
if (!university) { res.status(404).json({ success: false, message: UNIVERSITY_ERRORS.NOT_FOUND }); return; }

  res.status(409).json({ success: false, message: UNIVERSITY_ERRORS.CODE_EXISTS }); return;

await UniversityModel.update(Number(req.params.id), { code, name, address, phone, email, website, description, status });
```

### 5. `src/routes/university.routes.ts`
One flat router. Authorization on write routes only; read routes open to any authenticated user.

```typescript
import { Router } from 'express';
import { body, param, query } from 'express-validator';
import {
  createUniversity, updateUniversity, deleteUniversity,
  getUniversities, getUniversityDetail,
  createMajor, updateMajor, deleteMajor,
  getMajorsByUniversity, getMajorDetail,
} from '../controllers/university.controller';
import { authenticate, authorize } from '../middleware/auth.middleware';

const router = Router();

// ── Universities ──────────────────────────────────────────────
router.post('/', authenticate, authorize('ADMIN'),
  [
    param('id').optional().isInt({ gt: 0 }),
    body('code').trim().notEmpty(),
    body('name').trim().notEmpty(),
    body('email').optional().isEmail(),
    body('status').optional().isIn(['ACTIVE', 'INACTIVE']),
  ],
  createUniversity
);

router.put('/:id', authenticate, authorize('ADMIN'),
  [
    param('id').isInt({ gt: 0 }).withMessage('ID must be a positive integer'),
    body('code').optional().trim().notEmpty(),
    body('email').optional().isEmail(),
    body('status').optional().isIn(['ACTIVE', 'INACTIVE']),
  ],
  updateUniversity
);

router.delete('/:id', authenticate, authorize('ADMIN'),
  [param('id').isInt({ gt: 0 })],
  deleteUniversity
);

router.get('/',
  [
    query('page').optional().isInt({ gt: 0 }),
    query('limit').optional().isInt({ gt: 0 }),
  ],
  getUniversities
);

router.get('/:id',
  [param('id').isInt({ gt: 0 })],
  getUniversityDetail
);

// ── Majors ────────────────────────────────────────────────────
router.post('/:universityId/majors', authenticate, authorize('ADMIN'),
  [
    param('universityId').isInt({ gt: 0 }),
    body('code').trim().notEmpty(),
    body('name').trim().notEmpty(),
    body('admission_combinations_id').isInt({ gt: 0 }),
    body('min_score').optional().isDecimal(),
    body('status').optional().isIn(['ACTIVE', 'INACTIVE']),
  ],
  createMajor
);

router.put('/:universityId/majors/:majorId', authenticate, authorize('ADMIN'),
  [
    param('majorId').isInt({ gt: 0 }),
    body('code').optional().trim().notEmpty(),
    body('email').optional().isEmail(),
    body('admission_combinations_id').optional().isInt({ gt: 0 }),
    body('min_score').optional().isDecimal(),
    body('status').optional().isIn(['ACTIVE', 'INACTIVE']),
  ],
  updateMajor
);

router.delete('/:universityId/majors/:majorId', authenticate, authorize('ADMIN'),
  [param('majorId').isInt({ gt: 0 })],
  deleteMajor
);

router.get('/:universityId/majors',
  [
    param('universityId').isInt({ gt: 0 }),
    query('page').optional().isInt({ gt: 0 }),
    query('limit').optional().isInt({ gt: 0 }),
  ],
  getMajorsByUniversity
);

router.get('/:universityId/majors/:majorId',
  [param('majorId').isInt({ gt: 0 })],
  getMajorDetail
);

export default router;
```
